## 2. How did you use AI as a teammate?
I used Claude on this project to help write tests, fix bugs and write tests to verify. For the high/low bug, AI suggested to reverse the labels, stop casting the secret int to a string and then remove the unnecessary try/catch block.

One example of AI correctly suggesting a bug fix was the identification of the attempts counter starting at 1 instead of 0. This prevented the user from receiving the number of tries as per the difficulty level. I verified the correctness by running the game, as well as having the AI write a unit test which passed successfully. 

Another bug was the new_game button not working correctly. AI correctly identified that the game status and history were not being reset in the new_game logic. After AI made the edits, I verified by running the game and having AI write a unit test for this logic.

## 3. Debugging and testing your fixes

I decided the bugs were fixed when I ran the game and stepped through the same process which produced the bug in the first place. I also had the AI generate unit tests for each bug and reviewed them to assure they are covering several use cases.

For example, the test_new_game_restores_playable_status() generated by AI simulates a populated game status struct and then the behavior of the game when the new_game button is pressed. The test asserts that all of the fields are reset after being pressed. It shows how the game stores all of its dynamic data in one place.

AI helped design all of the unit tests after fixing several bugs. Previously this work would have taken an hour or more given the various use cases and scenarios. I didn't rely on AI to help me understand any tests as they all appeared self-explanatory.

## 4. What did you learn about Streamlit and state?

Streamlit transforms structured data into a reusable and dynamic web app. Through interaction on the webpage, data is stored and processed by the streamlit backend and then displayed on the webpage with the code dictating it's behavior. The session state is purged when the app is closed or manually within the app by pressing the new_game button which is programmed to reset the state.
